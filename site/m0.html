What was built

Key takeaways

What this milestone established

  • Structure precedes measurement. Every later milestone has a defined home for code (src/esm2_perf/, scripts/), data (results/raw/), tests, and docs.
  • Honesty is built in from the start. The README's "does not claim" section and the plan's review gates exist before a single number is produced.
  • Reproducibility over convenience. requirements.txt is intentionally unpinned for portability; exact installed versions are captured separately in the environment artifact (Milestone 1) so any number can be tied to a known stack.
  • Repo hygiene rules. No weights, venvs, caches, secrets, or large traces are ever committed; work is committed locally and never pushed.
