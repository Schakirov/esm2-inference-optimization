Milestone 1 · Environment & GPU check

A self-contained, gracefully-degrading environment probe that records the exact stack benchmarks run on, captured to a small diffable text artifact. The first executable, GPU-touching step.

What was built

scripts/00_check_environment.py

  • Prints system info, PyTorch/CUDA details (version, availability, cuDNN), per-device GPU properties (name, compute capability, total/allocated/reserved memory, SM count), bf16 support, selected package versions, and an nvidia-smi CSV snapshot.
  • Fails gracefully — a missing torch, no CUDA device, or absent nvidia-smi produce readable messages rather than tracebacks, so the repo is inspectable on CPU-only hosts.
  • --output writes the report to a file (creating parent dirs); saved to results/raw/environment.txt.

Key takeaways

What this milestone established

  • The GPU path is viable. CUDA is available and reports the L4 with bf16 hardware support, confirming the benchmark path for later milestones works.
  • Reproducibility anchor exists. One diffable artifact ties every future number to a known Python / PyTorch / CUDA / GPU / package combination.
  • The ~22 GiB vs 23034 MiB gap is expected. PyTorch reports memory visible to the runtime; nvidia-smi reports board total. The difference is driver/runtime reservation, not an error.
  • The L4 is an inference GPU. Ada generation, 72 W class, power-efficient — which is exactly why ESM2 embedding throughput on it is a realistic, interesting target.
Limitation at this gate

No ESM2 weights were downloaded yet, so the transformers ESM2 code path was not yet validated; a model-load failure could still surface in Milestone 2 (it did not).
