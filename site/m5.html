Milestone 5 · torch.compile & static shapes

Test whether graph compilation helps ESM2 encoder inference on the L4, and under what shape regime — being careful to separate one-time compile cost from steady-state speed.

What this milestone will do

Plan

  • Compile vs eager. Benchmark torch.compile'd inference against the eager baseline across the same shape matrix.
  • Warmup honesty. Explicitly separate compilation/warmup time from steady-state timed iterations — counting compile time as inference time is a classic misleading artifact.
  • Static vs dynamic shapes. Examine recompilation behavior when sequence length / batch size vary, and whether fixing shapes (padding to buckets) avoids recompiles.
Planned outputs

Artifacts

  • scripts/04_bench_compile.py — compile-vs-eager benchmark.
  • A timestamped compile CSV under results/raw/.
No overclaiming

If torch.compile does not beat eager PyTorch on this model/hardware, that result is reported as measured and the reason documented in docs/LIMITATIONS.md.
