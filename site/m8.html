Key takeaways

The report is regenerated from data, not hand-maintained, so it cannot silently drift. The summarizer selects the richest CSV per experiment (most rows), so a --quick smoke run never shadows the full matrix. Consolidated story: length-sorted batching is the cheapest real win (~1.7×), torch.compile is a real 1.3×–2.2× with a ~28 s/shape cold start, and the Triton kernel is correct and fastest-in-shape but negligible end-to-end — reported as honestly as the wins.
