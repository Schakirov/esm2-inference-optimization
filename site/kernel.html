The Triton masked-mean-pooling kernel, line by line

The full kernel from Milestone 6 (src/esm2_perf/triton_pooling.py), reproduced here with extensive comments. It fuses the masked mean pooling step — multiply by mask, sum over the sequence axis, divide by the token count — into a single pass that never writes the (B, T, H) product back to GPU memory.

Mental model

How the work is split

The kernel is launched as a 2-D grid of independent programs of shape (B, ⌈H / BLOCK_H⌉). Program (pid_b, pid_h) owns exactly one batch row and one BLOCK_H-wide slice of the 1280-dim hidden vector, and is solely responsible for out[pid_b, pid_h·BLOCK_H : +BLOCK_H]. Inside, it walks the sequence axis T in BLOCK_T-sized chunks, keeping a running fp32 sum and token count in registers — so the big (B, T, H) tensor is read once, streaming, and the masked sum + divide happen on-chip.

The kernel

Two equivalent versions of the same kernel — identical results, pick your reading level. Full · in use is the committed source: explicit strides and runtime T/H, so it stays general and compiles once for all shapes. Simplified · readable is a shorter variant that bakes in the contiguous layout and marks T/H as constexpr (trade-offs noted under it).

# ===========================================================================
# esm2_perf/triton_pooling.py
# Masked mean pooling over real amino-acid tokens, as ONE fused GPU kernel.
# ===========================================================================
#
# What it computes, for every sequence b and hidden channel h:
#
#     out[b, h]  =  ( Σ_t  mask[b,t] · hidden[b,t,h] )  /  max( Σ_t mask[b,t], 1 )
#
#   hidden : (B, T, H)   per-token encoder outputs (T includes <cls>/<eos> + padding)
#   mask   : (B, T)      1.0 = real amino-acid token, 0.0 = padding / special token
#   out    : (B, H)      one pooled embedding per sequence  (H = 1280 for ESM2-650M)
#
# Why hand-write it: do the multiply, the sum over T, and the divide in a SINGLE
# pass that never materialises the (B, T, H) product in GPU memory. The PyTorch
# one-liner `(hidden * mask).sum(1)` writes that whole tensor to HBM and reads it
# back; pooling is memory-bound, so removing that round-trip is the entire win.

import torch

# Triton is a GPU-only dependency. Import it lazily so that importing this module
# on a CPU-only machine (e.g. for the non-GPU unit tests, or `import esm2_perf`)
# does not raise — HAS_TRITON gates the kernel and its launcher.
try:
    import triton
    import triton.language as tl
    HAS_TRITON = True
except ImportError:
    HAS_TRITON = False


@triton.jit                              # compiled to a GPU kernel on first call, then cached
def _masked_mean_pool_kernel(
    hidden_ptr, mask_ptr, out_ptr,        # base GPU addresses of the three tensors
    T, H,                                 # runtime sizes: sequence length, hidden size
    stride_hb, stride_ht, stride_hh,      # hidden strides: elements to step per b / t / h
    stride_mb, stride_mt,                 # mask strides:   per b / t
    stride_ob, stride_oh,                 # out strides:    per b / h
    BLOCK_T: tl.constexpr,                # compile-time tile sizes. constexpr => baked into
    BLOCK_H: tl.constexpr,                #   the kernel, so loops unroll & registers are sized
):
    # ---- which slice of the output does THIS program instance compute? ----
    # The grid is 2-D: (B, ceil(H / BLOCK_H)). Each (pid_b, pid_h) pair is an
    # independent program that produces out[pid_b, pid_h*BLOCK_H : +BLOCK_H].
    # There is no cross-program communication — every output element has exactly
    # one owner, so no atomics or locks are needed.
    pid_b = tl.program_id(0)              # this program's batch row
    pid_h = tl.program_id(1)              # this program's BLOCK_H-wide hidden-column slice

    # The hidden columns this program writes, e.g. [256, 257, ... 511].
    # The final column block may run past H (1280 = 5×256 is exact here, but the
    # kernel must stay correct for any H), so carry a validity mask alongside.
    offs_h = pid_h * BLOCK_H + tl.arange(0, BLOCK_H)   # (BLOCK_H,) column indices
    h_in   = offs_h < H                                # (BLOCK_H,) True where the column is real

    # ---- accumulators live in registers, in fp32, for the whole reduction ----
    # acc   = running Σ mask·hidden for these BLOCK_H columns
    # count = running Σ mask for this row (same for every column → a scalar)
    # Accumulating in fp32 regardless of the input dtype is exactly why this
    # kernel is MORE accurate than bf16 PyTorch pooling (err ~1e-8 vs ~1e-3).
    acc   = tl.zeros((BLOCK_H,), dtype=tl.float32)
    count = tl.zeros((1,),       dtype=tl.float32)

    # ---- stream over the sequence axis T, one BLOCK_T chunk at a time ----
    # This is the reduction loop. T can be hundreds of tokens; we never hold the
    # whole row at once, just a BLOCK_T×BLOCK_H tile per iteration.
    for t0 in range(0, T, BLOCK_T):
        offs_t = t0 + tl.arange(0, BLOCK_T)        # (BLOCK_T,) token indices in this chunk
        t_in   = offs_t < T                            # (BLOCK_T,) True where token is real (not tail pad)

        # Load the mask for these tokens. Lanes past T read 0.0 (other=0.0), so they
        # contribute nothing to either the weighted sum or the count.
        m = tl.load(
            mask_ptr + pid_b * stride_mb + offs_t * stride_mt,
            mask=t_in, other=0.0,
        ).to(tl.float32)                               # (BLOCK_T,)

        # Build the address of every element in the (BLOCK_T, BLOCK_H) hidden tile.
        # Broadcasting a column vector of token offsets against a row vector of
        # hidden offsets yields a 2-D grid of per-element addresses:
        #   element (t, h) lives at  base + b·stride_b + t·stride_t + h·stride_h
        h_ptrs = (
            hidden_ptr
            + pid_b * stride_hb
            + offs_t[:, None] * stride_ht           # (BLOCK_T, 1)
            + offs_h[None, :] * stride_hh           # (1, BLOCK_H)
        )
        # Load the tile. The 2-D mask zeroes out lanes outside the real (T, H) region
        # (ragged tail in either dimension). Upcast to fp32 for the math.
        tile = tl.load(
            h_ptrs, mask=t_in[:, None] & h_in[None, :], other=0.0,
        ).to(tl.float32)                               # (BLOCK_T, BLOCK_H)

        # Weight each token's row by its mask value, then reduce over the token axis
        # (axis 0). Padding tokens have m = 0, so they add nothing — that is the
        # "masked" in masked mean. This keeps the running sum entirely on-chip.
        acc   += tl.sum(tile * m[:, None], axis=0)     # (BLOCK_H,)
        count += tl.sum(m, axis=0)                     # scalar held in a (1,) tensor

    # ---- finalize: divide the masked sum by the token count to get the mean ----
    # Clamp the denominator to >= 1 so a sequence with NO real tokens yields
    # 0 / 1 = 0 (a clean zero vector) instead of 0 / 0 = NaN. This matches the
    # PyTorch reference (which clamps the count the same way) bit-for-bit on the
    # edge case.
    denom = tl.maximum(count, 1.0)                  # (1,)
    out   = acc / denom                                # (BLOCK_H,) broadcast divide

    # Write the pooled values back to out[pid_b, offs_h], masking off any columns
    # that ran past H so we never scribble outside the tensor.
    tl.store(out_ptr + pid_b * stride_ob + offs_h * stride_oh, out, mask=h_in)


def triton_masked_mean_pool(
    hidden_states, mask, *,
    out_dtype=torch.float32,              # return fp32 by default; caller may ask for bf16/fp16
    block_t=128, block_h=256,           # tile sizes (fixed here — NOT autotuned per shape)
):
    """Host-side launcher: validate, make contiguous, launch the 2-D grid."""
    # --- guard rails: this kernel only makes sense on a CUDA tensor of rank 3 ---
    if not HAS_TRITON:
        raise RuntimeError("triton is not available")
    if not hidden_states.is_cuda:
        raise ValueError("requires CUDA tensors")
    if hidden_states.dim() != 3:
        raise ValueError("expected (B, T, H)")

    B, T, H = hidden_states.shape

    # The kernel indexes with explicit strides, so the layout must be the one we
    # pass. .contiguous() guarantees the standard row-major strides; it's a no-op
    # if the tensor is already contiguous (the common case straight off the model).
    hidden_states = hidden_states.contiguous()
    # Cast the mask to fp32 once on the host so the kernel can accept an int/bool/
    # float attention_mask uniformly without dtype branches inside the kernel.
    mask = mask.to(torch.float32).contiguous()

    # Output buffer. Always fp32 — it holds the fp32-accumulated result; we cast to
    # out_dtype only at the very end (below).
    out = torch.empty((B, H), device=hidden_states.device, dtype=torch.float32)

    # 2-D launch grid: B rows  ×  ceil(H / BLOCK_H) hidden-column blocks.
    # triton.cdiv is ceiling-division so a non-multiple H still covers every column.
    grid = (B, triton.cdiv(H, block_h))

    # Launch. We pass raw .stride(i) values so the kernel's pointer arithmetic works
    # for any (contiguous) layout without assuming H- or T-major hard-coded steps.
    _masked_mean_pool_kernel[grid](
        hidden_states, mask, out,
        T, H,
        hidden_states.stride(0), hidden_states.stride(1), hidden_states.stride(2),
        mask.stride(0), mask.stride(1),
        out.stride(0), out.stride(1),
        BLOCK_T=block_t, BLOCK_H=block_h,
    )
    return out.to(out_dtype)               # fp32 by default; cast down only if asked

# ---------------------------------------------------------------------------
# Simplified variant — same math, fewer moving parts, identical results.
# Two deliberate shortcuts vs the full version (costs noted below the code):
#   * No stride arguments. Addresses assume the standard CONTIGUOUS layout:
#       row b at  b·T·H,  token t at  +t·H,  channel h at  +h.
#     The launcher's .contiguous() call is what makes that assumption safe.
#   * T and H are tl.constexpr (compile-time), so the loop can fully unroll —
#     at the cost of recompiling the kernel once per distinct (T, H) shape.
# ---------------------------------------------------------------------------
if HAS_TRITON:

    @triton.jit
    def _masked_mean_pool_kernel(
        hidden,                           # (B, T, H) contiguous — base pointer
        mask,                             # (B, T)    contiguous — base pointer
        out,                              # (B, H)    contiguous — base pointer
        T: tl.constexpr,                  # sequence length (constexpr => loop unrolls per shape)
        H: tl.constexpr,                  # hidden size = 1280
        BLOCK_T: tl.constexpr,            # tokens handled per loop iteration
        BLOCK_H: tl.constexpr,            # hidden channels handled by this program
    ):
        b = tl.program_id(0)                # this program's batch row
        h = tl.program_id(1) * BLOCK_H + tl.arange(0, BLOCK_H)   # its hidden columns
        h_ok = h < H                                       # guard the final column block

        acc   = tl.zeros((BLOCK_H,), dtype=tl.float32)     # Σ mask·hidden — fp32 registers
        count = tl.full((), 0.0, tl.float32)            # Σ mask — a 0-D fp32 scalar

        for t0 in range(0, T, BLOCK_T):           # stream over the sequence axis
            t = t0 + tl.arange(0, BLOCK_T)
            t_ok = t < T                                   # guard the ragged tail

            # mask[b, t]:  row b starts at b·T, token t at +t. Pad lanes read 0.0.
            m = tl.load(mask + b * T + t, mask=t_ok, other=0.0).to(tl.float32)
            # hidden[b, t, h] = b·T·H + t·H + h, broadcast to a (BLOCK_T, BLOCK_H) tile.
            x = tl.load(
                hidden + b * T * H + t[:, None] * H + h[None, :],
                mask=t_ok[:, None] & h_ok[None, :],
                other=0.0,
            ).to(tl.float32)

            acc   += tl.sum(x * m[:, None], axis=0)        # masked sum over tokens
            count += tl.sum(m, axis=0)                    # token count

        y = acc / tl.maximum(count, 1.0)               # mean; clamp => all-pad row → 0, not NaN
        tl.store(out + b * H + h, y, mask=h_ok)            # Triton casts fp32 → out.dtype on store


def triton_masked_mean_pool(
    hidden_states, mask, *,
    out_dtype=torch.float32, block_t=128, block_h=256,
):
    if not HAS_TRITON:
        raise RuntimeError("triton is not available")
    if not hidden_states.is_cuda:
        raise ValueError("requires CUDA tensors")
    if hidden_states.dim() != 3:
        raise ValueError("expected (B, T, H)")

    hidden_states = hidden_states.contiguous()        # the kernel assumes this exact layout
    mask = mask.to(torch.float32).contiguous()

    B, T, H = hidden_states.shape
    # Allocate straight in out_dtype: tl.store casts the fp32 result on the way out,
    # so there is no separate .to() or extra tensor at the end.
    out = torch.empty((B, H), device=hidden_states.device, dtype=out_dtype)

    _masked_mean_pool_kernel[(B, triton.cdiv(H, block_h))](
        hidden_states, mask, out,
        T, H, BLOCK_T=block_t, BLOCK_H=block_h,
    )
    return out

Shorter and equivalent — produces identical results. The two simplifications have costs:

  • Bakes in the contiguous layout. Dropping the stride arguments and hard-coding b·T·H + t·H + h only works because the launcher calls .contiguous(); it would be wrong on a transposed or strided view. The full version passes explicit strides and stays general.
  • T/H as constexpr. Lets the loop unroll, but the kernel recompiles once per distinct (T, H). Fine for fixed-shape benchmarking; a cost under variable-length traffic, where the full version's runtime T/H compile once and serve all shapes.
Three details worth dwelling on

Why fp32, why the clamp, why two masks

  • fp32 accumulation. The tile is upcast and summed in fp32 no matter the input dtype, so the result is essentially the fp32 reference (err ~1e-8) while bf16 PyTorch pooling sums in bf16 and carries ~1e-3. Faster and more accurate — but it means the kernel is not a bit-identical drop-in for the bf16 path.
  • tl.maximum(count, 1.0). An all-padding sequence has count = 0. Without the clamp that is 0 / 0 = NaN; with it, 0 / 1 = 0, a clean zero vector that matches the reference's edge-case behaviour.
  • The two load masks. t_in guards the ragged tail along the sequence axis and h_in the last hidden-column block; other=0.0 makes out-of-range lanes contribute nothing, so the kernel is correct for any T and H, not just multiples of the block sizes.
Honest framing

Correct and fast — but not a pipeline win

Read the results page for the caveats

This kernel is the fastest pooling path in every shape and near-exact, but pooling is ~0.08 ms of a 25–225 ms encoder forward, so it does not move end-to-end throughput. It is also not autotuned (fixed BLOCK_T/BLOCK_H). See Milestone 6 for the full benchmark, the honest baseline comparison (vs einsum, not just the naive path), and why the production recommendation is a one-line einsum rather than a bespoke kernel.
