Milestone 6 · Triton masked-mean-pooling kernel

Write one small custom Triton kernel that fuses the masked mean pooling step, and benchmark it honestly against the PyTorch implementation — validated for correctness against the Milestone 3 reference.

What this milestone will do

Plan

  • Fused pooling kernel. Implement masked mean pooling as a single Triton kernel (mask-aware reduction over the sequence axis) to avoid intermediate allocations and extra passes.
  • Correctness gate first. The kernel must match the fp32 eager reference from Milestone 3 within documented tolerances before any speed number is reported.
  • Honest benchmark. Compare against the PyTorch pooling path across the shape matrix; report where it wins, ties, or loses.
Planned outputs

Artifacts

  • src/esm2_perf/triton_pooling.py — the Triton kernel + Python wrapper.
  • Unit tests asserting numerical agreement with the reference pooling.
  • scripts/05_bench_triton_pooling.py + a timestamped CSV.
No overclaiming

The custom kernel is not claimed to be universally faster than PyTorch. The L4's Ada compute capability (8.9) and Triton 3.7.0 set the ceiling; results are reported as measured, wins and losses alike.
